Tutorial Completo para Colaboradores
Sistema ExpiWish de Análise de Feedback

1. Bem-vindo ao Sistema ExpiWish

Como colaborador do sistema ExpiWish, você tem acesso a uma poderosa ferramenta de análise de feedback que foi desenvolvida especificamente para ajudar profissionais de hotelaria a entender melhor seus hóspedes e melhorar continuamente a experiência oferecida. Este sistema utiliza inteligência artificial avançada para processar e analisar comentários dos hóspedes de forma automática e inteligente.

O ExpiWish não é apenas um sistema de coleta de dados - é uma plataforma completa que transforma feedback bruto em insights acionáveis, permitindo que você tome decisões informadas para melhorar a satisfação dos hóspedes e a operação do seu hotel.

O que você pode fazer com o ExpiWish:
  • Visualizar análises detalhadas de feedback de hóspedes
  • Importar dados de diferentes plataformas de avaliação
  • Acompanhar tendências de satisfação ao longo do tempo
  • Identificar problemas recorrentes e oportunidades de melhoria
  • Gerar relatórios para apresentação a gestores
  • Monitorar o impacto de ações implementadas

2. Primeiro Acesso e Login

2.1 Recebendo suas Credenciais

Suas credenciais de acesso foram criadas por um administrador do sistema e enviadas para seu email cadastrado. Você recebeu um email com seu usuário (que é seu endereço de email) e uma senha temporária que deve ser alterada no primeiro login.

Passos para primeiro login:
  1. Abra o email com suas credenciais
  2. Acesse o link do sistema ExpiWish fornecido
  3. Digite seu email como usuário
  4. Digite a senha temporária recebida
  5. Clique em "Entrar"
  6. O sistema solicitará alteração da senha - crie uma senha segura
  7. Confirme a nova senha
  8. Você será direcionado ao dashboard principal

2.2 Navegação Inicial

Após o login, você será direcionado automaticamente para o dashboard principal do seu hotel. Este dashboard é personalizado e mostra apenas dados relacionados ao hotel onde você trabalha, garantindo foco e relevância nas informações apresentadas.

2.3 Interface do Sistema

A interface do ExpiWish foi projetada para ser intuitiva e eficiente. No lado esquerdo da tela, você encontra o menu de navegação principal com todas as funcionalidades disponíveis. A área central mostra o conteúdo da seção selecionada, e há sempre informações contextuais para ajudar no uso de cada funcionalidade.

3. Compreendendo o Dashboard Principal

3.1 Visão Geral do Dashboard

O dashboard principal é sua central de informações e ponto de partida para todas as análises. Ele apresenta uma visão consolidada e atualizada da situação atual do seu hotel em termos de satisfação dos hóspedes, com dados organizados de forma clara e visual.

Na parte superior do dashboard, você encontra métricas principais como avaliação média atual, total de feedbacks processados, tendência geral (se a satisfação está melhorando ou piorando) e alertas importantes que requerem sua atenção.

3.2 Métricas Principais

  • Avaliação Média: A nota média atual do seu hotel, calculada com base em todos os feedbacks recentes
  • Total de Feedbacks: Quantidade total de comentários processados no período selecionado
  • Tendência: Indicador visual que mostra se a satisfação está subindo (verde), descendo (vermelho) ou estável (azul)
  • Última Atualização: Quando os dados foram atualizados pela última vez
  • Alertas: Notificações sobre situações que requerem atenção imediata

3.3 Gráficos e Visualizações

O dashboard apresenta diversos gráficos que ajudam a visualizar as informações de forma intuitiva:

  • Gráfico de Tendência Temporal: Mostra como a avaliação evoluiu ao longo do tempo
  • Distribuição por Categorias: Exibe performance em diferentes aspectos (limpeza, atendimento, localização, etc.)
  • Volume de Feedbacks: Quantidade de comentários recebidos por período
  • Análise de Sentimento: Proporção de comentários positivos, neutros e negativos
Dica de Uso: Dedique alguns minutos todas as manhãs para revisar o dashboard. Isso te dará uma visão rápida do "estado de saúde" da satisfação dos hóspedes e permitirá identificar rapidamente se há algo que precisa de atenção imediata.

4. Sistema de Filtros e Personalização

4.1 Compreendendo os Filtros

Uma das funcionalidades mais poderosas do sistema são os filtros, que permitem segmentar e analisar os dados de acordo com suas necessidades específicas. Os filtros estão disponíveis na maioria das telas e permitem refinar a visualização dos dados.

4.2 Tipos de Filtros Disponíveis

  • Período: Selecione datas específicas, últimos 7 dias, 30 dias, 3 meses, etc.
  • Fonte: Filtre por origem do feedback (Booking.com, Google, Airbnb, etc.)
  • Categoria: Foque em aspectos específicos como limpeza, atendimento, localização
  • Avaliação: Visualize apenas feedbacks com determinadas notas (ex: apenas 1-2 estrelas)
  • Idioma: Analise comentários em idiomas específicos
  • Sentimento: Filtre por comentários positivos, negativos ou neutros

4.3 Como Usar os Filtros Efetivamente

Fluxo de trabalho recomendado para análise:
  1. Visão geral: Comece sempre sem filtros para ter a visão completa
  2. Identifique padrões: Note tendências ou picos de satisfação/insatisfação
  3. Investigue problemas: Use filtros para focar em períodos ou categorias problemáticas
  4. Analise melhorias: Compare períodos antes/depois de implementar mudanças
  5. Documente insights: Anote descobertas importantes para ações futuras

4.4 Salvando Configurações Favoritas

O sistema lembra das suas configurações de filtro mais utilizadas, facilitando análises recorrentes. Você pode salvar combinações específicas de filtros para relatórios regulares ou análises especializadas.

5. Interação com Gráficos e Dados

5.1 Gráficos Interativos

Todos os gráficos no sistema são interativos, permitindo uma exploração mais detalhada dos dados. Você pode clicar, navegar e obter informações específicas de cada ponto nos gráficos.

5.2 Funcionalidades dos Gráficos

  • Hover para detalhes: Passe o mouse sobre pontos para ver informações específicas
  • Zoom: Clique e arraste para aumentar o zoom em períodos específicos
  • Clique para drill-down: Clique em pontos para ver detalhes dos feedbacks daquele período
  • Legendas interativas: Clique nas legendas para ocultar/mostrar séries de dados
  • Exportação: Todos os gráficos podem ser exportados como imagem para relatórios

5.3 Análise de Tendências

Use os gráficos de tendência para identificar padrões temporais importantes:

  • Sazonalidade na satisfação dos hóspedes
  • Impacto de eventos especiais ou mudanças operacionais
  • Dias da semana com melhor/pior performance
  • Efeitos de ações corretivas implementadas
Dica Avançada: Quando identificar uma queda ou pico na satisfação, use o drill-down para ler os comentários específicos daquele período. Isso ajuda a entender exatamente o que causou a mudança e como agir.

6. Importação e Gestão de Dados

6.1 Seção "Importar Dados"

A funcionalidade de importação permite que você carregue dados de feedback de diferentes fontes e plataformas, centralizando toda a informação no sistema ExpiWish para análise unificada.

6.2 Formatos Suportados

O sistema aceita diferentes formatos de arquivo e fontes de dados:

  • Planilhas Excel (.xlsx): Formato mais comum para exportações de plataformas
  • CSV: Formato universal que funciona com qualquer plataforma
  • JSON: Para integrações técnicas mais avançadas
  • APIs diretas: Algumas plataformas podem ser conectadas automaticamente

6.3 Processo de Importação

Passos para importar dados:
  1. Acesse "Importar Dados" no menu lateral
  2. Escolha o tipo de fonte (Booking.com, Google, Airbnb, etc.)
  3. Selecione o arquivo no seu computador
  4. Aguarde o sistema processar e validar os dados
  5. Revise o preview dos dados que serão importados
  6. Configure mapeamento de campos se necessário
  7. Confirme a importação
  8. Aguarde o processamento completo
  9. Verifique se os dados apareceram no dashboard

6.4 Mapeamento de Campos

Diferentes plataformas organizam dados de formas distintas. O sistema ExpiWish possui um assistente inteligente que ajuda a mapear os campos corretamente:

  • Data do feedback: Quando o comentário foi feito
  • Avaliação/nota: A pontuação dada pelo hóspede
  • Comentário: O texto do feedback
  • Nome do hóspede: Identificação (quando disponível)
  • Categoria do quarto: Tipo de acomodação
  • Plataforma: Onde o comentário foi originalmente postado

6.5 Monitoramento da Importação

Durante e após a importação, você pode acompanhar o progresso e verificar se houve problemas:

  • Status em tempo real do processamento
  • Número de registros processados vs. total
  • Alertas sobre dados que não puderam ser processados
  • Log detalhado de erros (se houver)
  • Resumo final da importação
Importante: Sempre verifique a qualidade dos dados após a importação. Se notar inconsistências, entre em contato com o suporte técnico para ajustar os mapeamentos ou corrigir problemas na fonte dos dados.

7. Análise Detalhada de Feedback

7.1 Seção "Análise"

A seção de análise é onde você mergulha profundamente nos feedbacks dos hóspedes. Aqui, a inteligência artificial do sistema processa todos os comentários e apresenta insights organizados e acionáveis.

7.2 Categorização Automática

O sistema utiliza IA para categorizar automaticamente todos os feedbacks em áreas relevantes:

  • Limpeza: Comentários sobre higiene de quartos, banheiros, áreas comuns
  • Atendimento: Experiências com staff, recepção, concierge
  • Localização: Conveniência, acesso a transportes, proximidade de atrações
  • Instalações: Academia, piscina, restaurante, Wi-Fi
  • Custo-benefício: Percepção de valor pelo preço pago
  • Conforto: Qualidade do sono, ruído, temperatura
  • Café da manhã: Qualidade, variedade, serviço

7.3 Análise de Sentimento

Cada comentário é analisado para determinar o sentimento geral:

  • Positivo: Elogios, experiências satisfatórias
  • Neutro: Comentários factuais sem conotação emocional
  • Negativo: Reclamações, problemas, insatisfação
  • Misto: Comentários que contêm tanto aspectos positivos quanto negativos

7.4 Identificação de Temas Recorrentes

A IA identifica temas que aparecem frequentemente nos comentários, ajudando você a priorizar ações:

  • Problemas mais mencionados
  • Aspectos mais elogiados
  • Tendências emergentes
  • Comparações com períodos anteriores

7.5 Sugestões de Ação

Com base na análise dos dados, o sistema oferece sugestões práticas de ações que podem ser implementadas para melhorar a satisfação dos hóspedes. Essas sugestões são baseadas em boas práticas da indústria hoteleira e padrões identificados nos dados.

Como usar a análise detalhada:
  1. Revise o resumo geral: Comece com a visão overview da análise
  2. Identifique problemas críticos: Foque em categorias com maior insatisfação
  3. Leia comentários específicos: Entenda detalhes dos problemas
  4. Priorize ações: Liste melhorias por impacto e facilidade de implementação
  5. Monitore resultados: Acompanhe se as ações geram melhoria

8. Histórico e Acompanhamento Temporal

8.1 Seção "Histórico"

O histórico mantém um registro completo de todas as análises, importações e mudanças ao longo do tempo, permitindo acompanhar a evolução da satisfação dos hóspedes e o impacto de ações implementadas.

8.2 Análise de Evolução

Use o histórico para:

  • Comparar performance entre diferentes períodos
  • Identificar sazonalidades e padrões cíclicos
  • Avaliar o impacto de mudanças implementadas
  • Detectar tendências de longo prazo
  • Criar relatórios de progresso para gestores

8.3 Marcos e Eventos

Você pode marcar eventos importantes no histórico (como reformas, mudanças de política, treinamentos) para correlacionar com mudanças na satisfação dos hóspedes.

8.4 Exportação de Relatórios Históricos

Todos os dados históricos podem ser exportados para criação de relatórios executivos, apresentações ou análises externas mais aprofundadas.

9. Dados Não Identificados e Limpeza

9.1 Seção "Não Identificados"

Esta seção mostra dados que o sistema não conseguiu processar completamente durante a importação, geralmente devido a problemas de formatação ou informações incompletas.

9.2 Tipos de Problemas Comuns

  • Datas inválidas: Formatos de data não reconhecidos
  • Avaliações fora do padrão: Notas em escalas diferentes do esperado
  • Comentários corrompidos: Texto com caracteres especiais problemáticos
  • Campos obrigatórios vazios: Informações essenciais ausentes
  • Duplicatas: Mesmo feedback importado múltiplas vezes

9.3 Processo de Limpeza

Como limpar dados não identificados:
  1. Revise cada item na lista de não identificados
  2. Identifique o tipo de problema
  3. Corrija os dados diretamente no sistema quando possível
  4. Para problemas complexos, exporte e corrija na fonte
  5. Re-importe dados corrigidos
  6. Verifique se agora aparecem corretamente nas análises

9.4 Prevenção de Problemas

Para minimizar dados não identificados em futuras importações:

  • Use sempre formatos padronizados para datas
  • Verifique escalas de avaliação antes da exportação
  • Limpe dados na fonte antes de exportar
  • Use modelos fornecidos pelo sistema quando disponíveis
  • Teste importações com pequenos conjuntos de dados primeiro

10. Configurações Pessoais e Preferências

10.1 Seção "Configurações"

Nas configurações pessoais, você pode personalizar sua experiência no sistema e configurar informações da sua conta.

10.2 Configuração de API de IA

Se você possui uma chave de API própria para análise inteligente, pode configurá-la nas configurações. Isso permite análises mais avançadas e personalizadas.

Configurando API de IA:
  1. Acesse "Configurações" no menu lateral
  2. Vá para a aba "Configurações da API"
  3. Insira sua chave de API (deve começar com 'sk-')
  4. Clique em "Salvar Configuração"
  5. Aguarde a validação da chave
  6. A partir de agora, suas análises usarão sua própria API

10.3 Alteração de Senha

Por segurança, altere sua senha regularmente:

  • Use senhas com pelo menos 8 caracteres
  • Inclua letras maiúsculas e minúsculas
  • Adicione números e símbolos
  • Evite informações pessoais óbvias
  • Não compartilhe sua senha com outros

10.4 Preferências de Notificação

Configure como e quando deseja receber notificações sobre:

  • Novos feedbacks críticos
  • Conclusão de importações
  • Alertas de tendências negativas
  • Relatórios automáticos periódicos

11. Fluxo de Trabalho Diário Recomendado

11.1 Rotina Matinal (10-15 minutos)

Checklist diário:
  1. Revisar dashboard principal: Verificar métricas gerais e alertas
  2. Analisar feedbacks recentes: Ler comentários das últimas 24h
  3. Identificar issues urgentes: Problemas que precisam ação imediata
  4. Verificar tendências: Notar mudanças em relação aos dias anteriores
  5. Planejar ações do dia: Definir prioridades baseadas nos insights

11.2 Análise Semanal (30-45 minutos)

Uma vez por semana, dedique mais tempo para análise aprofundada:

  • Revisar tendências da semana
  • Comparar com semana anterior
  • Analisar categorias específicas em detalhes
  • Identificar padrões por dia da semana
  • Planejar ações para a próxima semana
  • Preparar relatório para gestores

11.3 Importação de Dados

Estabeleça uma rotina regular para importação:

  • Diariamente: Para hotéis com alto volume de feedback
  • 2-3 vezes por semana: Para volume médio
  • Semanalmente: Para hotéis menores ou períodos de baixa ocupação
Dica de Produtividade: Configure alertas automáticos para ser notificado quando há feedbacks negativos críticos. Isso permite resposta rápida a problemas sérios antes que afetem mais hóspedes.

12. Interpretação de Dados e Tomada de Decisão

12.1 Como Ler as Métricas

Compreender as métricas é essencial para tomar decisões informadas:

  • Avaliação média: Foque na tendência, não apenas no número absoluto
  • Volume de feedback: Mais comentários = mais oportunidades de melhoria
  • Distribuição de sentimentos: Procure padrões, não apenas proporções
  • Categorias problemáticas: Priorize por impacto na experiência geral

12.2 Priorizando Ações

Nem todos os problemas têm a mesma prioridade. Use esta matriz para decidir:

  • Alta prioridade: Problemas frequentes com alto impacto na satisfação
  • Média prioridade: Issues específicos que afetam alguns hóspedes
  • Baixa prioridade: Melhorias nice-to-have que podem aguardar
  • Ação imediata: Problemas de segurança ou que podem viralizar

12.3 Medindo Sucesso

Use o sistema para medir o impacto das suas ações:

  • Compare métricas antes/depois de implementar mudanças
  • Monitore se problemas específicos diminuíram
  • Verifique se melhorias geraram mais elogios
  • Acompanhe trends de longo prazo

13. Colaboração e Comunicação

13.1 Compartilhando Insights

Use o sistema para criar relatórios para diferentes stakeholders:

  • Para gerentes: Foque em métricas principais e ROI de ações
  • Para equipe operacional: Destaque problemas específicos e soluções
  • Para marketing: Compartilhe pontos fortes para comunicação

Elementos essenciais de um bom relatório:

  • Timeline para implementação

14. Troubleshooting e Problemas Comuns

14.1 Problemas de Importação

Problemas mais comuns e soluções:
  • Arquivo não reconhecido: Verifique se está no formato correto (.xlsx, .csv)
  • Dados duplicados: O sistema detecta automaticamente, mas verifique as datas
  • Mapeamento incorreto: Revise se os campos estão alinhados corretamente
  • Caracteres especiais: Salve planilhas com codificação UTF-8

14.2 Problemas de Performance

Se o sistema estiver lento:

  • Reduza o período de análise para datasets menores
  • Use filtros para focar em dados específicos
  • Limpe cache do navegador regularmente
  • Verifique sua conexão de internet

14.3 Dúvidas sobre Dados

Se os dados parecem incorretos:

  • Verifique os filtros aplicados
  • Confirme o período selecionado
  • Revise se todas as fontes foram importadas
  • Compare com dados da fonte original
